6、使用

到此为止，环境就配置完成了，如何使用呢？

  • 这里以 SpringCloud 微服务为例，每个微服务都有自己的端口，设有如下微服务

    • Student 服务：port=2001，RequestMapping=/stu
    • Department 服务：port=2002，RequestMapping=/dept

  • 微服务中的 Nacos、Sentinel、MySQL 服务器地址就可以直接使用对应 docker 容器的地址了

  • 再在 nginx.conf 的 server 块添加配置，令 /stu 转发到 2001 端口，/dept 转发到 2002 端口即可
